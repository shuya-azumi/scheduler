# 面接日程調整サービス データ設計（MVP）

**作成日**：2026-06-07
**対象**：Firebase **Realtime Database**（講義で習得済みの push / set / update / onChildAdded / onValue で全実装可能）
**方針**：チャット形式の3者（求職者・エージェント・事業所）日程調整。候補枠を**ボタン**で出し、**クリック＝チャットに流れる**。音声AIは採用せず（技術難度・正確さの両面で不利）。
**スコープ**：MVPコアのみ。メール/LINE通知・カレンダー連携・ログイン本実装は後回し。

---

## 1. 全体ツリー（Realtime DB ＝ 1個のJSON）

```
rooms/
  {roomId}/                         ← 調整1件＝1ルーム（URLで共有）
    info/
      title: "山田太郎さん × オリーヴ介護 面接調整"
      status: "adjusting"           ← adjusting | confirmed
      confirmedSlotId: ""           ← 確定したら枠IDが入る
      createdAt: 1718000000000
    members/
      {memberId}/                   ← 参加者（3者）
        name: "山田太郎"
        role: "candidate"           ← candidate | agent | facility
    slots/
      {slotId}/                     ← エージェントが出す候補枠
        label: "6/16(月) 13:00〜14:00"
        start: "2026-06-16T13:00"
        end:   "2026-06-16T14:00"
    votes/
      {slotId}/
        {memberId}: "ok"            ← ok | ng | maybe（1人1枠に1つ・上書き）
    messages/
      {messageId}/                  ← チャットの流れ（追記専用）
        name: "山田太郎"
        type: "vote"                ← text | vote | confirm | system
        text: "6/16 13:00 に ⭕ しました"
        slotId: "slot_2"            ← type=vote/confirm のとき
        createdAt: 1718000000000
```

## 2. 各ノードの役割

| ノード | 役割 | 主な操作 |
|---|---|---|
| `info` | ルームの状態（調整中／確定） | `set` / `update` |
| `members` | 誰が参加しているか・役割 | `push`（参加時1回） |
| `slots` | 候補日時のリスト | `push`（枠追加） |
| `votes` | 「どの枠に誰が○×か」の最新状態 | **`set`（上書き）** |
| `messages` | チャットの会話ログ | **`push`（追記）** |

## 3. 設計の肝：`votes` と `messages` を分ける

- **`votes`** … 「今この枠に誰が○か」という*最新状態*。○→×に変えたら上書きしたいので `votes/{slotId}/{memberId}` を `set`。→ 「全員○の枠」を即計算できる。
- **`messages`** … 「○しました」という*履歴*。消さず積み上げる → `push`（講義チャットと同じ）。

**ボタン1クリック＝2つ書き込む**（これが「クリックするとチャットに流れる」の正体）：
```
[6/16 13:00 に ⭕] クリック
  ① votes/{slotId}/{自分のid} を "ok" に set   ← 状態更新
  ② messages に push                            ← チャットに「⭕しました」が流れる
```

## 4. 4つの基本操作（push / set / update のみ）

| やりたいこと | 書き込み先 | メソッド |
|---|---|---|
| ① ルーム参加 | `members` | `push` |
| ② 候補枠を出す（エージェント） | `slots` | `push` |
| ③ 枠に○×する | `votes/{slotId}/{自分id}` ＋ `messages` | `set` ＋ `push` |
| ④ 確定する（エージェント） | `info`（status/confirmedSlotId）＋ `messages` | `update` ＋ `push` |

## 5. 画面への反映（習得済みの機能）

- **チャット欄** → `messages` を `onChildAdded`（新着が下に積まれる＝講義サンプルそのまま）
- **候補枠パネル** → `slots` と `votes` を `onValue`（誰かが○×したら枠表示を作り直す）

## 6. 「全員○の枠」ハイライト計算

`onValue` で `votes` 更新のたび、各枠について：
```js
const slotVotes = votes[slotId] || {};
const allOk = memberIds.every(id => slotVotes[id] === "ok");
if (allOk) { /* 緑ハイライト＝「ここで確定できます」 */ }
```
エージェントはハイライト枠の「確定」を押すだけ → 伝言ゲームが1画面に集約。

## 7. ルームへの入り方（MVP）

ログイン本実装はせず **URLで共有**：`index.html?room=room_abc123`
開いた人が「名前＋役割」を選ぶ → `members` に `push`。
